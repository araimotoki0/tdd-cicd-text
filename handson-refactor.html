改善編：Refactorフェーズ

テストが通ったコードをより良くする

Refactorとは？

📘 定義

Refactor（リファクタリング）とは、
外部から見た動作を変えずに、内部のコード構造を改善することです。

🎯 Refactorの目的

  • • コードの可読性を高める
  • • 保守しやすくする
  • • バグを減らす
  • • 将来の機能追加を容易にする

⚠️ 重要なルール

Refactorはテストがグリーン（成功）の状態でのみ行います。
テストが失敗している状態でRefactorしてはいけません。
なぜなら、変更によってバグが混入したかどうか判断できないからです。

REFACTOR：コードを改善する

基礎編で作ったCalculatorクラスを改善します

現在のコード

基礎編で作成したCalculatorクラスを見てみましょう。

<?php

namespace App\Services;

class Calculator
{
    /**
     * 2つの数を足す
     */
    public function add($a, $b)
    {
        return $a + $b;
    }
}

🤔 改善できるポイント

  • ❌ 型ヒントがない（どんな型を受け取るか不明確）
  • ❌ 戻り値の型が宣言されていない
  • ❌ ドキュメントコメントが簡素
  • ✅ でもテストは通っている！
1

テストが通っていることを確認

Refactorの前に、必ずテストが成功していることを確認します。

📝 実行するコマンド:

php artisan test tests/Unit/CalculatorTest.php

✅ 期待される結果:

  PASS  Tests\Unit\CalculatorTest
  ✓ it can add two numbers

  Tests:    1 passed
  Duration: 0.03s

グリーン（成功）であることを確認したら、Refactorを開始できます。

2

型ヒントを追加する

引数と戻り値に型ヒントを追加して、コードを明確にします。

📚 型ヒントとは？

PHPでは、関数の引数や戻り値の型を明示的に宣言できます。
これにより、間違った型が渡された場合にエラーで気付けます。
また、IDEの補完機能も効くようになり、開発が効率化します。

src/app/Services/Calculator.php を以下のように修正してください： 

<?php

namespace App\Services;

class Calculator
{
    /**
     * 2つの数を足す
     */
    public function add(int|float $a, int|float $b): int|float
    {
        return $a + $b;
    }
}

📖 変更点の解説:

int|float $a, int|float $b

引数の型を指定。int|floatは「整数または小数」という意味（Union型）。
PHP 8.0以降で使えます。

: int|float

戻り値の型を指定。この関数は整数または小数を返します。

💡 Refactorのポイント

型ヒントを追加しても、外部から見た動作は変わりません。
これがRefactorの重要な特徴です。機能は同じまま、コードの品質だけを高めています。

3

テストを再実行

Refactor後は必ずテストを実行します。

📝 実行するコマンド:

php artisan test tests/Unit/CalculatorTest.php

✅ 期待される結果:

  PASS  Tests\Unit\CalculatorTest
  ✓ it can add two numbers

  Tests:    1 passed
  Duration: 0.03s

引き続きグリーン（成功）のままであれば、Refactorは成功です！

🎉

Refactor成功！

型ヒントを追加することで、コードがより明確になりました。

📚 学んだこと:

  • ✓ Refactorはテストがグリーンの時だけ行う
  • ✓ 外部の動作を変えずに内部構造を改善
  • ✓ Refactor後は必ずテストを実行
  • ✓ 型ヒントでコードを明確にできる
4

さらなる改善（オプション）

ドキュメントコメントをより詳細にしてみましょう。 

<?php

namespace App\Services;

class Calculator
{
    /**
     * 2つの数値を加算する
     *
     * @param int|float $a 第1の加算数
     * @param int|float $b 第2の加算数
     * @return int|float 加算結果
     */
    public function add(int|float $a, int|float $b): int|float
    {
        return $a + $b;
    }
}

変更後、再度テストを実行します。

php artisan test tests/Unit/CalculatorTest.php

引き続きグリーンであれば完璧です！
このように、Refactorは小さな改善を繰り返すプロセスです。
