🎯 Replitとは？

💻 概要

Replitは、
ブラウザ上でコードを書いて実行できるオンラインIDEです。

主な特徴

  • • 環境構築が簡単：テンプレートからすぐに開始
  • • リアルタイム共同編集：Google Docsのように同時編集可能
  • • Multiplayer機能：最大4人まで同時にコーディング
  • • カーソル色分け：誰がどこを編集しているか一目瞭然

🎯 今日やること

Replitを使って、チームでFizzBuzzをTDDで実装します。
モブプログラミング形式で、全員で1つのコードを作り上げましょう！

⚠️ Composerの選択について

Shellタブ（右上の「+」→ Shell）で以下を実行：

🚨 PHPバージョンエラーが出た場合

「This version of PHPUnit requires PHP >= 8.2」エラーが出た場合：

rm -rf vendor composer.lock && composer require --dev phpunit/phpunit:^10

→ PHPUnit 10を使うことで解決します

✅ 確認ポイント

  • • vendor/ フォルダが作成される
  • • composer.json が作成される
  • • composer.lock が作成される

📖 FizzBuzz TDDチュートリアル

赤 → 緑 → リファクタリング のサイクルを体験

📝 FizzBuzzの仕様

1から100までの数字を出力するプログラムを書いてください。
ただし：

  • • 3の倍数のときは、数字の代わりに「Fizz」を出力
  • • 5の倍数のときは、数字の代わりに「Buzz」を出力
  • • 3と5の両方の倍数のときは「FizzBuzz」を出力

👨‍🏫 講師と一緒に進める部分（STEP 1〜6）

STEP 1: テストを書く（赤）

tests/FizzBuzzTest.php は既に準備済み

./vendor/bin/phpunit tests/

→ 赤（テスト失敗）

STEP 2: テストを通す最小限のコード（緑）

function fizzbuzz($n)
{
    return "1";  // とりあえず "1" を返す
}

→ 緑（テスト成功）

STEP 3: 次のテスト追加（赤）

public function test_returns_2_when_given_2()
{
    $this->assertEquals("2", fizzbuzz(2));
}

→ 赤（"1"が返るので失敗）

STEP 4: テストを通す（緑）

function fizzbuzz($n)
{
    return (string)$n;  // 数字を文字列で返す
}

→ 緑（両方のテスト成功）

STEP 5: 3の倍数のテスト追加（赤）

public function test_returns_Fizz_when_given_3()
{
    $this->assertEquals("Fizz", fizzbuzz(3));
}

→ 赤（"3"が返るので失敗）

STEP 6: 3の倍数でFizzを返す（緑）

function fizzbuzz($n)
{
    if ($n % 3 === 0) {
        return "Fizz";
    }
    return (string)$n;
}

→ 緑（全テスト成功）

💡 $n % 3 は「nを3で割った余り」。余りが0なら3の倍数！

🔥 ここからはチームで挑戦！

STEP 6までのパターンを参考に、残りを自分たちで実装しよう！

📋 やることリスト:

STEP 7-8: 5の倍数で「Buzz」を返すテストと実装

STEP 9-10: 15の倍数で「FizzBuzz」を返すテストと実装

STEP 11: 1から100まで出力（オプション）

💡 ヒント

  • • 5の倍数は $n % 5 === 0 でチェック
  • • 15の倍数の判定は順番が重要！（最初にチェックしないと...？）
  • • 困ったらチームで相談。ナビゲーターがアドバイス！
🙈 模範解答（STEP 7〜11）- まずは見ないでチャレンジ！

⚠️ 注意

まずはチームで考えてみましょう！どうしても分からないときだけ参考に。

STEP 7-8: 5の倍数でBuzz

// テスト追加
public function test_returns_Buzz_when_given_5()
{
    $this->assertEquals("Buzz", fizzbuzz(5));
}

// 実装
function fizzbuzz($n)
{
    if ($n % 3 === 0) {
        return "Fizz";
    }
    if ($n % 5 === 0) {
        return "Buzz";
    }
    return (string)$n;
}

STEP 9-10: 15の倍数でFizzBuzz

// テスト追加
public function test_returns_FizzBuzz_when_given_15()
{
    $this->assertEquals("FizzBuzz", fizzbuzz(15));
}

// 実装（15を最初にチェック！）
function fizzbuzz($n)
{
    if ($n % 15 === 0) {
        return "FizzBuzz";
    }
    if ($n % 3 === 0) {
        return "Fizz";
    }
    if ($n % 5 === 0) {
        return "Buzz";
    }
    return (string)$n;
}

💡 ポイント

15の倍数を最初にチェック！15は3でも5でも割り切れるので、後にすると"Fizz"になってしまう。

STEP 11: 1から100まで出力

// FizzBuzz.php の末尾に追加
for ($i = 1; $i <= 100; $i++) {
    echo fizzbuzz($i) . "\n";
}

🔧 トラブルシューティング

PHPUnitが動かない

原因：インストール失敗

解決策：composer require --dev phpunit/phpunit を再実行

Permission denied エラー

原因：権限エラー

解決策：chmod +x vendor/bin/phpunit を実行

招待リンクが機能しない

原因：リンク期限切れ

解決策：新しいリンクを生成

メンバーが参加できない

原因：Replitアカウント未作成

解決策：アカウントを作成してから再度リンクにアクセス

📝 よく使うコマンド

# テスト実行
./vendor/bin/phpunit tests/

# 特定のテストだけ実行
./vendor/bin/phpunit tests/FizzBuzzTest.php --filter test_returns_Fizz_when_given_3

# PHPUnitバージョン確認
./vendor/bin/phpunit --version
