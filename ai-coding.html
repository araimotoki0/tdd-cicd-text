AI Coding入門：Gemini CLI

ターミナルからAIでコーディングを効率化

Gemini CLIとは？

🤖 概要

Gemini CLIは、Googleが提供する
ターミナルから直接AIを使えるオープンソースツールです。

✨ 主な特徴

  • • 無料で使える：60リクエスト/分、1,000リクエスト/日
  • • 大容量：100万トークンのコンテキストウィンドウ
  • • ファイル操作：コードの読み取り・編集をAIに任せられる
  • • シェル連携：コマンド実行もAIがサポート

✅ 前提条件

環境構築ページでGemini CLIのインストールが完了していること。
まだの方は Mac環境構築 または Windows環境構築 を先に完了してください。

ハンズオン：Gemini CLIを使ってみよう

基本的な操作を体験します

1

初回起動と認証

ターミナルでGemini CLIを起動します。

📝 実行するコマンド:

gemini

⚠️ 重要：個人アカウントでログインしてください

無料で使えるのは個人のGoogleアカウントのみです。

会社や学校のGoogle Workspaceアカウントでは無料版が使えません。
Workspaceで使うには有料プラン（Standard $19〜/月）が必要になります。
公式FAQ →

🎨 初回起動時の流れ:

1. テーマ選択

CLIの見た目のテーマを選びます。
カーソルキー（↑↓）で選択し、Enterで決定。

2. 認証方法の選択

「Login with Google」を選択してEnter。
ブラウザが開き、個人のGoogleアカウントでログインします。

3. 認証完了

「許可」をクリックすると、ターミナルに戻ります。
これで準備完了です！

✅ 成功のサイン:

プロンプト（入力待ち状態）が表示されれば成功です。
❯ のような記号が表示されます。

💡 認証の切り替え・ログアウト

別のGoogleアカウントに切り替えたい場合や、ログアウトしたい場合は
/auth コマンドを使います。 

❯ /auth

認証方法の選択画面が再表示され、別のアカウントでログインし直せます。

💡 モデルの切り替え

使用するAIモデルを変更したい場合は
/model コマンドを使います。 

❯ /model

2

日本語設定（GEMINI.md）

デフォルトでは英語で応答されることがあります。
GEMINI.mdファイルを編集して、常に日本語で応答させましょう。

📖 GEMINI.mdとは？

Gemini CLIが自動で読み込む設定ファイルです。
AIへの指示を書いておくと、毎回のプロンプトに自動で適用されます。

📁 配置場所と優先順位:

1. グローバル設定（全プロジェクト共通）

~/.gemini/GEMINI.md

← 日本語設定はここに書くのがおすすめ！

2. プロジェクト設定

プロジェクトルート/GEMINI.md

プロジェクト固有の指示を追加

※ 複数のGEMINI.mdがある場合、全て統合されてAIに送信されます。
公式ドキュメント →

📝 グローバル設定に追記:

~/.gemini/GEMINI.md を開き、以下を追記：

Always respond in Japanese.

✅ 効果:

グローバル設定に書いておけば、どのプロジェクトでも
AIが常に日本語で応答するようになります。

💡 プロジェクト固有の設定例

プロジェクトルートのGEMINI.mdには、そのプロジェクト特有の情報を書きます：

  • • プロジェクトの概要・目的
  • • 使用している技術スタック
  • • コーディング規約・命名規則
  • • ディレクトリ構成の説明

Laravelでアプリを作る前に

データベース連携アプリを作るための基礎知識を確認しましょう

⚠️ 重要：Laravelコマンドの実行場所

Laravelの php artisan コマンドは、Dockerコンテナの中で実行します。 

# まずコンテナに入る
docker compose exec app bash

# コンテナ内でartisanコマンドを実行
php artisan make:model Todo -m

※ コンテナから出るときは exit を入力

🏗️ MVCアーキテクチャとは？

LaravelはMVC（Model-View-Controller）というパターンでコードを整理します。
役割ごとにファイルを分けることで、コードが整理され保守しやすくなります。

📦 Model

データの管理

  • • DBのテーブルと対応
  • • データの取得・保存
  • • ビジネスロジック
app/Models/Todo.php

🎨 View

画面の表示

  • • HTMLの生成
  • • ユーザーへの表示
  • • Bladeテンプレート
resources/views/todo.blade.php

🎮 Controller

処理の制御

  • • リクエストの受付
  • • Modelを呼び出す
  • • Viewを返す
app/Http/Controllers/TodoController.php

🔄 処理の流れ

ユーザー → Controller → Model → DB → Model → Controller → View → ユーザー

🗄️ LaravelでDBを作る流れ

1

Model と Migration を作成

1つのコマンドでModelとテーブル定義を同時に作れます

php artisan make:model Todo -m

-m オプションで migration（テーブル定義）も一緒に作成

2

Migration ファイルを編集

テーブルのカラム（列）を定義します

// database/migrations/xxxx_create_todos_table.php
public function up()
{
    Schema::create('todos', function (Blueprint $table) {
        $table->id();
        $table->string('title');        // タイトル
        $table->boolean('completed')->default(false); // 完了フラグ
        $table->timestamps();        // 作成日時・更新日時
    });
}
3

Migration を実行

定義したテーブルを実際にDBに作成します

php artisan migrate
4

Controller を作成

CRUD操作を行うControllerを作成します

php artisan make:controller TodoController --resource

--resource オプションでCRUD用のメソッドが自動生成されます

💡 AIに任せよう！

これらの手順はAIが全てやってくれます。
「TodoアプリをLaravelで作って」と伝えれば、Model・Migration・Controllerを自動で作成・編集してくれます。
