AI Coding入門：Antigravity CLI

ターミナルから「複数のAIエージェント」でコーディングを効率化

Google Antigravity が提供する次世代のターミナルAI（コマンド名：agy）

なぜ Antigravity CLI に移行したのか

📢 結論：Gemini CLI は提供終了します

これまで本講座で使ってきた Gemini CLI は、 2026年6月18日 をもって 無料ユーザー・Google AI Pro／Ultra ユーザー向けのリクエスト処理を終了します。

その後継として Google が公式に案内しているのが、本ページで学ぶ Antigravity CLI です。
（※ Standard／Enterprise ライセンスをもつ企業ユーザーは、引き続き Gemini CLI のサポートを受けられます）

🕒 これまでの流れ

2025年11月18日：Gemini 3 と Google Antigravity を発表

Google が新モデル Gemini 3 と、それを土台にした エージェント型開発プラットフォーム Google Antigravity（IDE）を同時公開。 AIを「ツール」ではなく「能動的な開発パートナー」として位置づけ直しました。

2026年5月19日：Antigravity CLI を一般公開

Antigravity プラットフォームの一部として、ターミナル版の Antigravity CLI をリリース。あわせて Antigravity 2.0（デスクトップアプリ・CLI・SDK）が登場しました。

2026年6月18日：Gemini CLI（無料／Pro／Ultra）が提供終了

開発リソースを後継製品に集約するため、Gemini CLI の一般向け提供を終了。 以降は Antigravity CLI に一本化されます。

💡 移行の理由

Google の公式アナウンスでは、移行の背景がこう説明されています。

  • ① 開発のスタイルが「マルチエージェント」へ進化した
    2025年初頭の「AIに1つずつ質問する」使い方から、 複数のエージェントが互いに連携し、作業を分担して複雑な問題を解く使い方へとニーズが変化しました。
  • ② 製品を1つに集約して品質を高めるため
    複数のツールに力を分散させるのではなく、 今日のマルチエージェント時代に合わせた単一の製品に開発リソースを集中することにしました。
  • ③ Gemini CLI の資産はそのまま引き継げる
    Agent Skills・Hooks・Subagents・Extensions（Antigravity では「プラグイン」）といった 主要機能が Antigravity CLI に継承されるため、移行コストを抑えられます。

出典： Google Developers Blog「Transitioning Gemini CLI to Antigravity CLI」、 Google Antigravity 公式サイト、 Wikipedia: Google Antigravity

Antigravity CLIとは？

🚀 概要

Antigravity CLI は、Googleが提供する
ターミナルから直接AIエージェントを動かせるコマンドラインツールです。

エージェント型開発プラットフォーム「Google Antigravity」のターミナル版にあたり、 裏側では Gemini 3 が動いています。 コマンド名は agy です。

✨ 主な特徴

  • • Go製の軽量バイナリ：Node.js / npm は不要。高速・低オーバーヘッド
  • • マルチエージェント：複数のサブエージェントを並列に動かしてタスクを分担
  • • ファイル操作・コマンド実行：許可を取りながらコードの読み書きやコマンド実行を代行
  • • 無料枠あり：無料の Google アカウントでも利用可能
  • • 資産の継承：Gemini CLI の Skills / Hooks / Subagents / 拡張（プラグイン）に対応

⚠️ 無料枠を使うときの注意

Antigravity CLI は複数のサブエージェントを並列で起動できるため、 1つのコマンドが裏で複数のワーカーを動かし、無料枠の消費が想像より早くなることがあります。
消費量は /usage コマンドで確認できます。

✅ 前提条件

ターミナル（Mac はターミナル.app、Windows は PowerShell など）が使えること。
個人の Google アカウントを用意しておいてください。

ハンズオン：Antigravity CLIを使ってみよう

インストールから基本操作までを体験します

0

インストール

公式インストーラを実行します。環境を自動判別し、agy バイナリを配置します。

🍎 macOS / Linux:

curl -fsSL https://antigravity.google/cli/install.sh | bash

バイナリは ~/.local/bin/ に配置されます。

🪟 Windows（PowerShell）:

irm https://antigravity.google/cli/install.ps1 | iex

✅ インストール確認:

agy --version

バージョンが表示されれば成功です。
あとから更新したいときは agy update を実行します。

1

初回起動と認証

ターミナルで Antigravity CLI を起動します。

📝 実行するコマンド:

agy

🎨 初回起動時の流れ:

1. 認証方法の選択

ログイン方法の選択画面が出ます。
「Google でサインイン（Google OAuth）」を選んで Enter。

2. ブラウザでログイン

既定のブラウザが自動で開き、Google アカウントでログインします。
（SSH などリモート接続中の場合は、ブラウザの代わりに認証用URLが表示されます）

3. 認証完了

「許可」をクリックするとターミナルに戻ります。
認証情報はOSのキーチェーン（keyring）に保存され、次回からは自動でログインされます。

✅ 成功のサイン:

プロンプト（入力待ち状態）が表示されれば成功です。

💡 APIキーで使う場合（任意）

Google サインインの代わりに、APIキーを環境変数で渡すこともできます。 

export ANTIGRAVITY_API_KEY=あなたのAPIキー

💡 モデルの切り替え

セッション中に使用するモデルを変更したい場合は /model コマンドを使います。 

❯ /model
おすすめ

Gemini 3.5 Flash

高速・高性能。この研修ではこちらを使用

参考

Gemini 3.1 Pro

複雑な推論向け。Flashより応答が遅め

2

日本語設定（.antigravity.md）

デフォルトでは英語で応答されることがあります。
コンテキストファイルを編集して、常に日本語で応答させましょう。

📖 コンテキストファイルとは？

Antigravity CLI が自動で読み込む指示ファイルです。
AIへの指示を書いておくと、毎回のプロンプトに自動で適用されます。

📁 読み込まれるファイルと優先順位:

1. .antigravity.md（推奨）

Antigravity CLI 専用。最優先で読み込まれます

2. AGENTS.md

エージェント向けの共通標準ファイル

3. GEMINI.md（後方互換）

旧 Gemini CLI の設定もそのまま読み込めます

※ .antigravity.md と GEMINI.md が両方ある場合は .antigravity.md が優先されます。
※ いま読み込まれている設定ファイルは agy inspect で確認できます。

📝 設定例:

プロジェクトルートに .antigravity.md を作り、以下を記載：

💡 プロジェクト固有の設定例

コンテキストファイルには、そのプロジェクト特有の情報を書いておくと便利です：

  • • プロジェクトの概要・目的
  • • 使用している技術スタック
  • • コーディング規約・命名規則
  • • ディレクトリ構成の説明

Laravelでアプリを作る前に

データベース連携アプリを作るための基礎知識を確認しましょう

⚠️ 重要：Laravelコマンドの実行場所

Laravelの php artisan コマンドは、Dockerコンテナの中で実行します。 

# まずコンテナに入る
docker compose exec app bash

# コンテナ内でartisanコマンドを実行
php artisan make:model Todo -m

※ コンテナから出るときは exit を入力

🏗️ MVCアーキテクチャとは？

LaravelはMVC（Model-View-Controller）というパターンでコードを整理します。
役割ごとにファイルを分けることで、コードが整理され保守しやすくなります。

📦 Model

データの管理

  • • DBのテーブルと対応
  • • データの取得・保存
  • • ビジネスロジック
app/Models/Todo.php

🎨 View

画面の表示

  • • HTMLの生成
  • • ユーザーへの表示
  • • Bladeテンプレート
resources/views/todo.blade.php

🎮 Controller

処理の制御

  • • リクエストの受付
  • • Modelを呼び出す
  • • Viewを返す
app/Http/Controllers/TodoController.php

🔄 処理の流れ

ユーザー → Controller → Model → DB → Model → Controller → View → ユーザー

🗄️ LaravelでDBを作る流れ

1

Model と Migration を作成

1つのコマンドでModelとテーブル定義を同時に作れます

php artisan make:model Todo -m

-m オプションで migration（テーブル定義）も一緒に作成

2

Migration ファイルを編集

テーブルのカラム（列）を定義します

// database/migrations/xxxx_create_todos_table.php
public function up()
{
    Schema::create('todos', function (Blueprint $table) {
        $table->id();
        $table->string('title');        // タイトル
        $table->boolean('completed')->default(false); // 完了フラグ
        $table->timestamps();        // 作成日時・更新日時
    });
}
3

Migration を実行

定義したテーブルを実際にDBに作成します

php artisan migrate
4

Controller を作成

CRUD操作を行うControllerを作成します

php artisan make:controller TodoController --resource

--resource オプションでCRUD用のメソッドが自動生成されます

💡 AIに任せよう！

これらの手順はAIが全てやってくれます。
「TodoアプリをLaravelで作って」と伝えれば、Model・Migration・Controllerを自動で作成・編集してくれます。

🛠️ スキル(Skills)を作ってみよう

発展編 よく使う作業手順をAIに覚えさせて再利用する

「スキル(Skills)」とは、よく使う作業手順や指示をひとまとめにして、いつでも呼び出せるようにする仕組みです。
毎回同じ説明を書かなくても、登録したスキルを呼ぶだけでAIが決まった手順で作業してくれます。

💡 スキルを使ったことがある人へ

すでにスキルを使ったことがある人がいれば、ぜひ AIを活用して新しいスキルを作ってみましょう。
「○○をするスキルを作って」とAIに頼めば、スキルの定義ファイルそのものをAIが書いてくれます。
